The "Brains" of the App

How it Works

Ever wondered how everything fits together? Company Bull is built using a few simple main parts that work together to keep everything running fast and safe. We use a set of "standard" tools so any developer can easily understand the code.

The Tech Stack

Frontend (The Part You See)

This is built with React. It’s what users see and interact with. It’s fast and smooth, so it feels like a real app, not just a website.

Backend (The Brain)

This is built with Node.js and Express. It’s the "hidden" part that handles all the heavy logic and processes requests from the frontend.

Database (The Memory)

We use MongoDB to store all your data securely. It remembers every employee, every task, and every cent spent.

State (The Context)

This is how we remember who is logged in and what they are allowed to see as they move through the app.

How Data Moves

Making sure data moves correctly is essential. Here is the 4-step path it follows:

1. ACTION

A user clicks a button or fills a form in the React app.

2. REQUEST

The app sends a "message" to the backend server with the data.

3. PROCESS

The backend server checks the data and does what’s needed.

4. SAVE

Everything is saved in the database so it’s ready for next time.

Security First

We take security seriously. Every time the app asks for data from the backend, the server checks who is asking. If they aren't authorized (like an employee trying to see the overall payroll), the server simply says "No."
