Module 28: The Talent Map

Skills Matrix

The Skills Matrix is the company's "Brainpower Map." It’s a powerful grid that shows exactly what every person is good at. From coding to project management, this tool helps you find the right expert for any project and identifies where the team might need more training.

01. The Skills vs. Staff Grid

The heart of this module is the **Mastery Table**. Every row is an employee, and every column is a skill. At the intersection, you’ll see a percentage and a **Mini Progress Bar**. This gives you an instant "Visual Score" of how much that person knows about a certain topic.

04. Engineering Blueprint

Behind the Scenes (Code)

How we built the competency map:

Skill Aggregation

We use flatMap() and new Set() to automatically pull every unique skill from hundreds of employee records into one single list.

Sticky Headers

We use sticky left-0 and bg-card in CSS to make sure the "Employee Name" column stays visible even when you scroll the huge table sideways.

Math for Averages

The Summary Cards use a .reduce() loop to add up all scores and divide them by the number of experts to find the "Team Average."

Grid Filtering

The search bar doesn't just hide employees—it hides **Skill Columns**! If you search for "Java," only skills related to Java will stay visible in the table.

// Finding unique skills automatically

05. How to use this page

Looking for an expert? Here’s the best way to leverage the Skills Matrix:

STEP 1: FILTER THE VIEW

Type a specific technology (like "React") into the search bar. This hides all other columns so you can focus on one skill at a time.

STEP 2: FIND THE GAPS

Look at the **Skill Summary Cards**. If an important skill has a low average (e.g., 50%), it might be time to organize a training workshop!

STEP 3: IDENTIFY LEADERS

Search for the green **Expert** labels. These are your senior leads who can help mentor others and review code on high-risk projects.
